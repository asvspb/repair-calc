### 0.2 Auth Bypass Strategy

**Problem:** AuthContext checks token via API call (`authApi.getCurrentUser()`). Simply injecting `token` + `refreshToken` into localStorage causes a failed API call → auth state becomes `isAuthenticated: false`.

**Solution:** Add a test-mode flag. In `AuthContext.tsx`, add an early return when a special localStorage key `test-auth-bypass` is set:
```ts
useEffect(() => {
  const checkAuth = async () => {
    // Test bypass: skip API calls
    if (localStorage.getItem('test-auth-bypass') === 'true') {
      setState({ user: { id: 'test-user', email: 'test@test.com', name: 'Test' }, isAuthenticated: true, isLoading: false, error: null });
      return;
    }
    // ... existing logic
  };
  checkAuth();
}, []);
```

In `e2e/fixtures.ts`, inject this key via `page.addInitScript`.
