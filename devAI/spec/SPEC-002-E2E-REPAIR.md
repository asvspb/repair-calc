## 1. Цель задачи

Восстановить работоспособность и стабильность набора E2E тестов (Playwright). На текущий момент 50 из 52 тестов падают из-за устаревших селекторов и несовместимости окружения.

---

## 2. Проблематика

1.  **Несовместимость Node.js**: Текущая версия Node (v12) не поддерживает современный Playwright (v1.58+), что вызывает синтаксические ошибки при запуске.
2.  **Хрупкие селекторы**: Большинство тестов полагаются на динамические или устаревшие CSS-классы/текст, которые изменились в процессе разработки v4.x.
3.  **Отсутствие CI-стабильности**: Тесты не дают достоверной информации о качестве кода, так как «всегда красные».

---

## 3. Требования к реализации

### 3.1 Окружение и инфраструктура
*   **ER-001**: Обновить версию Node.js до 18.x или 20.x в среде выполнения тестов.
*   **ER-002**: Убедиться, что `playwright.config.ts` содержит корректные порты (3993) и базовые URL.
*   **ER-003**: Настроить headless-режим для CI и UI-режим для отладки.

### 3.2 Селекторы и типизация
*   **ER-010**: Добавить атрибуты `data-testid` во все критические React-компоненты:
    *   Поля ввода геометрии (`geom-length`, `geom-width`, etc.)
    *   Кнопки управления объектами и проектами.
    *   Элементы списка работ и материалов.
*   **ER-011**: Обновить Page Object Model (POM) в `e2e/pages` для использования `data-testid` вместо CSS-селекторов.

### 3.3 Тестовое покрытие (Critical Path)
Необходимо восстановить прохождение следующих сценариев:
1.  **Auth**: Регистрация, логин, выход.
2.  **Projects**: Создание проекта, удаление, переключение.
3.  **Objects**: Создание объекта внутри проекта, редактирование.
4.  **Geometry**: Ввод размеров (Simple mode), проверка расчета площади и периметра.
5.  **Works**: Добавление работы из каталога, расчет стоимости материалов.
6.  **Export/Import**: Экспорт проекта в JSON и его последующий импорт.
