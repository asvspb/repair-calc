# Инструкция для проверки исправления бага с секциями

## Контекст
Исправлен баг, при котором в Расширенном режиме при добавлении секций и внесении данных о габаритах значения ранее введенных данных сбрасывались или произвольно изменялись.

## Как проверить

### 1. Откройте приложение
- Dev-сервер: **http://localhost:3993**
- Production (Docker): **http://localhost:3980**

### 2. Очистите кэш браузера
- **Ctrl+Shift+R** (Windows/Linux)
- **Cmd+Shift+R** (Mac)

### 3. Тестовый сценарий

#### Шаг 1: Создание комнаты
1. Создайте новую комнату
2. Переключите режим геометрии на **"Расширенный"** (Extended mode)

#### Шаг 2: Добавление первой секции
1. Нажмите "+ Добавить секцию"
2. Выберите форму: **Прямоугольник**
3. Введите габариты:
   - Длина: **5** м
   - Ширина: **4** м

#### Шаг 3: Добавление второй секции
1. Нажмите "+ Добавить секцию" еще раз
2. Выберите форму: **Трапеция**
3. Введите габариты:
   - Основание 1: **6** м
   - Основание 2: **4** м
   - Глубина: **5** м
   - Бок. сторона 1: **5** м
   - Бок. сторона 2: **5** м

#### Шаг 4: Проверка сохранения данных
1. Проверьте, что у **первой секции** (Прямоугольник):
   - Длина: **5** м (не изменилась)
   - Ширина: **4** м (не изменилась)

2. Проверьте, что у **второй секции** (Трапеция):
   - Основание 1: **6** м
   - Основание 2: **4** м
   - Глубина: **5** м
   - Бок. сторона 1: **5** м
   - Бок. сторона 2: **5** м

#### Шаг 5: Изменение размеров первой секции
1. Измените у первой секции Длину с **5** на **7** м
2. Проверьте, что у второй секции все значения **сохранились**

### 4. Ожидаемый результат

✅ **Правильно:**
- Все введенные значения сохраняются
- При изменении одной секции другие не меняются
- Метрики (площадь, периметр) пересчитываются корректно

❌ **Баг (если еще есть):**
- Значения в первой секции сбрасываются на 0
- Значения меняются произвольно
- Данные из разных секций перемешиваются
