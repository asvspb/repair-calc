## Задача 2: Импорт данных с созданием нового проекта

### Требования:
При импорте данных из JSON-файла автоматически создавать новый проект с возможностью указания названия.

### Логика работы:

1. Пользователь нажимает "Загрузить бэкап (JSON)"
2. Система парсит файл и проверяет валидность
3. Открывается диалог:
   - Поле ввода названия проекта
   - Дефолтное значение: "Импорт от DD.MM.YYYY"
   - Информация о количестве объектов
   - Кнопки "Отмена" и "Импортировать"
4. После подтверждения:
   - Создаётся новый проект с указанным названием
   - Все объекты из файла добавляются в этот проект
   - Проект добавляется к существующим проектам пользователя

### Диалог импорта:

```
┌─────────────────────────────────────────┐
│ Импорт данных                       [X] │
├─────────────────────────────────────────┤
│                                         │
│ Название нового проекта:                │
│ ┌─────────────────────────────────────┐ │
│ │ Импорт от 05.04.2026                │ │
│ └─────────────────────────────────────┘ │
│                                         │
│ ℹ️ Будет создан проект с N объектами    │
│                                         │
│              [Отмена]  [Импортировать]  │
└─────────────────────────────────────────┘
```

### Обработка edge-cases:

1. **Файл содержит несколько проектов:**
   - Все проекты добавляются к существующим
   - Каждый проект сохраняет свою структуру

2. **Файл содержит только rooms (старый формат):**
   - Создаётся один объект "Импортированный объект"
   - Все комнаты добавляются в этот объект

3. **Пустой файл или ошибка парсинга:**
   - Показать сообщение об ошибке
   - Не создавать проект
