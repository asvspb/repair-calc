# Bug Report: Дублирование отображения объектов ремонта

**Дата:** 2026-04-06
**Статус:** Подтверждён ✅
**Приоритет:** Средний
**Серьёзность:** Низкая (UI косметический баг)

---

## Описание бага

### Текущее поведение

В правой панели (RightSidebar) объект ремонта отображается **дважды**:

1. **В левой панели (ObjectSettings):**
   ```
   Объект ремонта
   └─ Дом (Подмосковье)  ← активный объект
   ```

2. **В правой панели (OtherObjectsSection):**
   ```
   Другие объекты
   ├─ Дом                ← ТОТ ЖЕ САМЫЙ объект (дублируется!)
   │   5 комнат
   ├─ Гараж
   │   2 комнаты
   └─ Дача
       4 комнаты
   ```

### Ожидаемое поведение

В секции "Другие объекты" должны отображаться **только другие объекты**, исключая активный.

**Правильный вариант:**
```
Другие объекты
├─ Гараж
│   2 комнаты
└─ Дача
    4 комнаты
```

## Анализ кода

### Где происходит баг

**Файл:** `src/components/layout/ObjectSettings.tsx`
**Функция:** `OtherObjectsSection` (строки 86-123)

### Логика фильтрации

```typescript
// Строка 95
const otherObjects = objects.filter(obj => obj.id !== activeObjectId);
```

**Проблема:** Фильтрация написана правильно, но `activeObjectId` может не совпадать с реальным активным объектом.

### Корневая причина

При инициализации приложения происходит рассинхронизация состояний:

1. **В `ProjectContext`:**
   - `activeObjectId` устанавливается автоматически при загрузке проекта
   - Значение по умолчанию: первый объект в проекте `objects[0]?.id`

2. **В `App.tsx`:**
   ```typescript
   // При загрузке контекста
   activeObjectId = objects[0]?.id  // например, 'obj-1'
   ```

3. **Проблема:**
   - Если `activeObjectId` равен `null` или не установлен корректно
   - Фильтр `obj.id !== activeObjectId` не сработает
   - Все объекты попадут в список "Другие объекты"

### Диагностика

Нужно проверить:

1. Значение `activeObjectId` в момент рендера
2. Инициализацию `activeObjectId` в `ProjectContext`
3. Синхронизацию между `activeObject` и `activeObjectId`

## Возможные решения

### Решение 1: Добавить защиту в OtherObjectsSection

```typescript
export function OtherObjectsSection({
  objects,
  activeObjectId,
  onObjectClick,
}: OtherObjectsProps) {
  // Если activeObjectId не установлен, не показываем ничего
  if (!activeObjectId) {
    return null;
  }

  // Фильтруем - показываем только другие объекты (не активный)
  const otherObjects = objects.filter(obj => obj.id !== activeObjectId);

  if (otherObjects.length === 0) {
    return null;
  }

  // ... остальной код
}
```

### Решение 2: Использовать activeObject вместо activeObjectId

```typescript
export function OtherObjectsSection({
  objects,
  activeObjectId,
  onObjectClick,
}: OtherObjectsProps) {
  // Получаем ID активного объекта из самого объекта
  const activeId = activeObjectId || objects[0]?.id;
  
  // Фильтруем - показываем только другие объекты (не активный)
  const otherObjects = objects.filter(obj => obj.id !== activeId);

  if (otherObjects.length === 0) {
    return null;
  }

  // ... остальной код
}
```

### Решение 3 (Рекомендуемое): Исправить инициализацию в ProjectContext

Проверить, что `activeObjectId` всегда корректно устанавливается при загрузке проекта.

**Файл:** `src/contexts/ProjectContext.tsx`

```typescript
// При загрузке проекта
const firstObject = project.objects?.[0];
setActiveObjectId(firstObject?.id || null);
```
