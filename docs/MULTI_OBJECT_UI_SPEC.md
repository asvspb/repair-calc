## 1. Контекст и проблема

### Текущее состояние

Бэкенд и фронтенд **полностью поддерживают** иерархию данных:
```
Project → Object[] → Room[]
```

Однако UI **отображает только первый объект**:
- В сайдбаре — селектор проектов, но нет селектора объектов
- Комнаты отображаются как плоский список
- Общая смета агрегирует данные только из `objects[0].rooms`

### Бизнес-кейс

**Зачем нужны несколько объектов в одном проекте?**

| Сценарий | Описание |
|----------|----------|
| **Квартира + гараж** | Основной объект ремонта и дополнительный (гараж, кладовка, балкон) |
| **Многоквартирный проект** | Инвестор ремонтирует несколько квартир в одном доме |
| **Дом + пристройки** | Частный дом с баней, гаражом, беседкой |
| **Коммерческий объект** | Офис + склад + торговый зал |
| **Поэтапный ремонт** | Разные объекты = разные этапы ремонта |

### Проблема

Пользователь вынужден создавать **отдельные проекты** для каждого объекта, что приводит к:
- Дублированию общих данных (город, настройки)
- Невозможности единой сметы по всем объектам
- Сложности сравнения затрат между объектами

## 2. Цели и задачи

### Цель

Предоставить пользователю возможность управлять несколькими объектами недвижимости в рамках одного проекта с единой сметой.

### Задачи

1. UI для создания/редактирования/удаления объектов
2. Навигация между объектами
3. Отображение объектов в общей смете
4. Импорт/экспорт объектов
5. Копирование объектов

## 3. Концепция UI

**Проект = Ремонтная кампания** (например, "Ремонт квартиры в Новосибирске")

### 5.4 Новые утилиты в projectObjects.ts

```typescript
/**
 * Создание нового пустого объекта
 */
export function createNewObject(
  projectId: string,
  data: { name: string; city?: string }
): ObjectData {
  return {
    id: `local-obj-${Date.now()}-${Math.random().toString(36).substr(2, 9)}`,
    projectId,
    name: data.name,
    city: data.city,
    rooms: [],
    sortOrder: 0,
  };
}

/**
 * Добавление объекта в проект
 */
export function addObjectToProject(
  project: ProjectData,
  newObject: ObjectData
): ProjectData {
  return {
    ...project,
    objects: [...(project.objects || []), newObject],
  };
}

/**
 * Копирование объекта со всеми комнатами
 */
export function copyObjectInProject(
  project: ProjectData,
  sourceObjectId: string
): { project: ProjectData; newObjectId: string } {
  const sourceObject = getObjectFromProject(project, sourceObjectId);
  if (!sourceObject) {
    throw new Error(`Object ${sourceObjectId} not found`);
  }

  // Создаём копию с новыми ID
  const newObjectId = `local-obj-${Date.now()}-${Math.random().toString(36).substr(2, 9)}`;
  const newRooms = sourceObject.rooms.map(room => ({
    ...room,
    id: `local-room-${Date.now()}-${Math.random().toString(36).substr(2, 9)}`,
  }));

  const newObject: ObjectData = {
    ...sourceObject,
    id: newObjectId,
    name: `${sourceObject.name} (копия)`,
    rooms: newRooms,
  };

  return {
    project: {
      ...project,
      objects: [...(project.objects || []), newObject],
    },
    newObjectId,
  };
}

/**
 * Обновление данных объекта
 */
export function updateObjectInProject(
  project: ProjectData,
  objectId: string,
  updates: Partial<ObjectData>
): ProjectData {
  if (!project.objects) return project;

  return {
    ...project,
    objects: project.objects.map(obj =>
      obj.id === objectId ? { ...obj, ...updates } : obj
    ),
  };
}

/**
 * Удаление объекта из проекта
 */
export function deleteObjectFromProject(
  project: ProjectData,
  objectId: string
): ProjectData {
  if (!project.objects) return project;

  // Нельзя удалить последний объект
  if (project.objects.length <= 1) {
    throw new Error('Cannot delete the last object in project');
  }

  return {
    ...project,
    objects: project.objects.filter(obj => obj.id !== objectId),
  };
}

/**
 * Переупорядочивание объектов
 */
export function reorderObjectsInProject(
  project: ProjectData,
  newOrder: string[] // массив ID объектов в новом порядке
): ProjectData {
  if (!project.objects) return project;

  const objectMap = new Map(project.objects.map(obj => [obj.id, obj]));
  const reorderedObjects = newOrder
    .map((id, index) => {
      const obj = objectMap.get(id);
      return obj ? { ...obj, sortOrder: index } : null;
    })
    .filter(Boolean) as ObjectData[];

  return {
    ...project,
    objects: reorderedObjects,
  };
}
```

## 8. Риски и ограничения

### Риски

| Риск | Вероятность | Влияние | Митигация |
|------|-------------|---------|-----------|
| Усложнение UI | Средняя | Среднее | Итеративный дизайн, user testing |
| Обратная совместимость | Низкая | Высокое | Миграция старых проектов уже реализована |
| Производительность при многих объектах | Низкая | Низкое | Виртуализация списков при необходимости |

### Ограничения

1. **Минимум один объект** — проект не может существовать без объектов. При удалении последнего объекта — автоматическое создание пустого.

2. **Нет drag-and-drop между объектами** — комнату нельзя перенести из одного объекта в другой (можно реализовать в будущем).

3. **Общая смета без фильтрации** — пока нет возможности посмотреть смету только по выбранным объектам.

### C. User Stories

1. **Как пользователь**, я хочу создать несколько объектов в одном проекте, чтобы считать смету по всему ремонту целиком.

2. **Как пользователь**, я хочу переключаться между объектами, чтобы работать с комнатами каждого объекта отдельно.

3. **Как пользователь**, я хочу видеть общую смету по всем объектам, чтобы понимать общие затраты на ремонт.

4. **Как пользователь**, я хочу копировать объект, чтобы быстро создать похожий (например, две одинаковые квартиры).

5. **Как пользователь**, я хочу удалять объекты, которые более не актуальны.
